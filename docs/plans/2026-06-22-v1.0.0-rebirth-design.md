# Design: v1.0.0 Rebirth ŌĆö Clean Open Source Launch

**Date:** 2026-06-22
**Status:** Approved
**Author:** Alexandr Narbaev

## Goal

Squash entire git history into a single initial commit (`v1.0.0`), restructure the project directories for cleaner open source presentation, add all standard open source documentation files, force-push to GitHub as a fresh project.

## Motivation

The project evolved through 36 internal iterations (v17 ŌåÆ v36.0.1) with accumulated history, plans, and research that should not be part of the public initial commit. A clean `v1.0.0` launch presents the project professionally, with proper open source documentation from day one.

## Design Decisions

### 1. Versioning: v1.0.0

Semantic versioning from scratch. The 36 iterations of internal development are acknowledged in README/CHANGELOG but not in git history.

### 2. Full History Erasure

- All old tags (v33.9, v33.10, v35.0) deleted from GitHub remote
- Single orphan commit as the new root
- Force push to `main`

## Git Operations (Sequence)

```
1. LOCAL BACKUP:    git tag archive/pre-v1.0.0 HEAD
2. DELETE REMOTE TAGS: git push github --delete v33.9 v33.10 v35.0
3. RESTRUCTURE:      mv lib/ src/lib/, mv modes/ src/modes/, etc.
4. UPDATE PATHS:     sed in all .sh files
5. CREATE DOCS:      CHANGELOG, .editorconfig, templates, etc.
6. ORPHAN COMMIT:    git checkout --orphan temp && git add -A
7. COMMIT:           git commit -m "v1.0.0: Initial public release"
8. TAG:              git tag v1.0.0
9. FORCE PUSH:       git push --force github main --tags
10. CLEANUP:         git branch -D main && git branch -m temp main
```
