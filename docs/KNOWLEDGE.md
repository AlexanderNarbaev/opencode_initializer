## 1. Что выбрать — быстрый гид (начинающим)

### Если у вас

| Ситуация | Решение | Команда |
|----------|---------|---------|
| Никогда не пользовались AI | Интерактивный режим, основы | `--interactive`, всё `n` кроме Node, Python, OpenCode |

## 3. RAG-инфраструктура (практикующим)

### Векторные БД — сравнение

| БД | RAM | Скорость | Сложность | Офлайн | Для чего | Документация |
|----|-----|----------|-----------|--------|----------|-------------|
| **ChromaDB** | 200MB | 50ms | Минимум | Да (ONNX) | Память агента | [docs.trychroma.com](https://docs.trychroma.com/) |
| Qdrant | 2GB+ | 10ms | Средне | Модели отдельно | Корпоративный RAG | [qdrant.tech](https://qdrant.tech/) |
| Weaviate | 4GB+ | 15ms | Высоко | Модели отдельно | Enterprise | [weaviate.io](https://weaviate.io/) |
| Milvus | 8GB+ | 5ms | Очень высоко | Нет | Big Data | [milvus.io](https://milvus.io/) |
| FAISS | in-memory | 1ms | Низко (без персист.) | Да | Эксперименты | [github](https://github.com/facebookresearch/faiss) |
| Pinecone | облако | 20ms | Низко | Нет | Serverless | [docs.pinecone.io](https://docs.pinecone.io/) |

### Почему ChromaDB для локальной разработки

- ONNX-модель (all-MiniLM-L6-v2, 79MB) — полностью офлайн
- Systemd-сервис — авто-запуск при логине
- 200MB RAM — подходит для ноутбуков
- Нативный бэкенд для Muninn

### Почему Qdrant для enterprise

- RBAC — изоляция по отделам
- Горизонтальное масштабирование — 100M+ векторов
- Фильтрация по метаданным
- Rust — производительность C++

### Гибридный поиск

Dense векторы (Qdrant) + Sparse векторы (BM25) = +35% точности. RAGFlow реализует обе стратегии.

## 5. Безопасность — полный анализ (практикующим / экспертам)

### Векторы атак на AI-агентов

| Вектор | Вероятность | Последствия | Защита |
|--------|------------|-------------|--------|
| Утечка API-ключей | Средняя | Компрометация аккаунта | secrets.env (chmod 600) |
| Утечка PII в промптах | Высокая | Нарушение GDPR/152-ФЗ | stranger-danger |
| Prompt injection | Средняя | Обход ограничений | stranger-danger |
| Деструктивные команды | Низкая | Потеря данных | damage-control (144 паттерна) |
| Чтение секретных файлов | Средняя | Утечка .env, .ssh | damage-control |
| SQL/Shell инъекции | Средняя | RCE, потеря данных | Trivy + Qodana |
| DoS (переполнение контекста) | Высокая | Рост затрат | dcp |
| Отказ провайдера | Средняя | Остановка работы | auto-fallback (6 providers) |

### 5 уровней защиты

```
Запрос пользователя
  → [1] secrets.env (chmod 600) — хранение ключей
  → [2] stranger-danger — фильтрация PII, инъекций
  → [3] damage-control (144 паттерна) — блокировка команд
  → [4] Trivy + Qodana — сканирование уязвимостей
  → [5] WSL2 hardening — изоляция ОС
  → Безопасное выполнение
```

## 6. CI/CD для AI-генерируемого кода (экспертам)

### Трёхэтапное ревью

```
Commit
  → [1] Автоматическое (linter, typecheck, tests) — 5 мин
  → [2] AI-ревью (n8n + Qwen3-Coder) — 15 мин
  → [3] Человек (mandatory для production) — по расписанию
```

### n8n AI Code Review Pipeline

```
GitLab Webhook (MR created)
  → GitLab API (получить diff)
  → RAGFlow (конвенции проекта)
  → Qwen3-Coder-14B (анализ diff)
  → GitLab API (опубликовать ревью)
```

**Статистика:** 80% стилевых замечаний + 45% логических ошибок до human review. 15% false positive.

### Trunk-Based Development для AI

- `main` — production (protected)
- `feature/*` — макс 2 дня, squash-merge
- Почему: AI-агенты работают лучше с маленькими PR

[n8n docs](https://docs.n8n.io/) · [GitLab CI docs](https://docs.gitlab.com/ee/ci/)

## 7. SGLang vs vLLM (экспертам)

| Критерий | SGLang | vLLM |
|----------|--------|------|
| Структурированная генерация | Нативная (JSON schema) | Ограниченная |
| RadixAttention | Да | Нет |
| PagedAttention | Нет | Да |
| Multi-node | Ручная | Ray-based |
| Лучше для | Single-GPU, constrained gen | Кластер, throughput |
| Документация | [sglang.readthedocs.io](https://sglang.readthedocs.io/) | [docs.vllm.ai](https://docs.vllm.ai/) |

### Проблема длинного контекста на 8GB

Принудительный 100K контекст на 8GB GPU → spill KV-кэша в RAM. PCIe 4.0 = 32 GB/s, GPU HBM = 1 TB/s → падение в 30x. Решение: RAG + нативный 32K контекст.

## 8. Архитектура памяти агента (экспертам)

```
СЕССИЯ (working memory)
  MemoryLayer MCP: remember, recall, reflect
  Agentic Tools MCP: задачи, прогресс
         │ commit
         ▼
ДОЛГОВРЕМЕННАЯ ПАМЯТЬ
  ChromaDB: векторы (384-мерные)
  Muninn: семантическая память сессий
  WAL: Markdown-лог состояния проекта
```

| Компонент | Тип памяти | Поиск | Персистентность |
|-----------|-----------|-------|----------------|
| MemoryLayer | Эпизодическая + семантическая | По смыслу | Да |
| Agentic Tools | Иерархическая (задачи) | По ID/статусу | Да |
| ChromaDB | Векторная | По близости | Да |
| Muninn | Сессионная | По содержимому | Да |
| WAL | Человекочитаемая | grep | Да |

[ChromaDB docs](https://docs.trychroma.com/) · [Qdrant docs](https://qdrant.tech/)
