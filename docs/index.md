# Ultimate Dev Machine Bootstrap v33.9

**Однокомандная настройка AI-усиленной dev-машины** — WSL2, Linux, macOS.

Один bash-скрипт (~2300 строк), который за 15–30 минут превращает чистую систему в полноценную среду разработки с AI-агентами, MCP-серверами, LSP, плагинами и GPU-рантаймами.

---

## Быстрый старт

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/AlexanderNarbaev/opencode_initializer/main/setup.sh) --full
```

**С API-ключами (рекомендуется):**
```bash
bash setup.sh --full \
  --deepseek-key "sk-..." \
  -k "opencode-api-key" \
  --xai-key "xai-..." \
  -n "Ваше Имя" \
  -e "email@example.com"
```

**Если не знаете что выбрать — интерактивный режим:**
```bash
bash setup.sh --interactive
```

> Скрипт сам задаст 16 вопросов: что ставить, что пропустить. Для новичков — идеально.

## Что такое OpenCode?

**OpenCode** — это CLI-инструмент (работает в терминале), который подключается к AI-моделям (DeepSeek, OpenAI, Anthropic и др.) и помогает писать код. Вы пишете задачу на естественном языке, а AI-агент её выполняет: ищет файлы, редактирует код, запускает тесты, коммитит в git.

**MCP (Model Context Protocol)** — это стандарт, позволяющий AI-агенту «видеть» ваши файлы, базы данных, браузер и другие инструменты. Без MCP агент работает «вслепую». С MCP он может читать ваш проект, запускать браузер, искать баги, анализировать код.

**LSP (Language Server Protocol)** — это то, что даёт автодополнение, подсветку ошибок и навигацию по коду в редакторе. Без LSP вы пишете код без подсказок.

**Этот скрипт устанавливает всё это** одной командой, с правильными настройками, без необходимости разбираться в каждом компоненте.

## Режимы работы

Скрипт поддерживает 10 режимов — от полной установки до точечного ремонта:

| Команда | Что делает | Когда использовать |
|---------|-----------|-------------------|
| `--full` | Полная установка всего | Первый запуск на чистой системе |
| `--reinit` | Переустановка инструментов, данные проекта сохраняются | Что-то сломалось, нужно переустановить |
| `--new <dir>` | Инициализация только нового проекта | Уже всё установлено, нужен новый проект |
| `--health` | Диагностика: 36+ проверок | Проверить что работает, а что нет |
| `--update` | Обновление инструментов до последних версий | Прошло время, нужно обновить |
| `--upgrade` | Полное обновление системы (apt+snap+SDKMAN+OMZ+rustup+npm+pip) | Капитальное обновление |
| `--interactive` | Пошаговый выбор компонентов (16 вопросов) | Не знаете что нужно — скрипт спросит |
| `--fix-config` | Перегенерировать opencode.json | Сломалась конфигурация OpenCode |
| `--fix-zshrc` | Починить .zshrc (дубликаты, PATH, P10k) | Терминал глючит |
| `--dry-run` | Показать что будет сделано (без реальных изменений) | Посмотреть план перед запуском |

**Важно:** скрипт отслеживает прогресс. Если упал на середине — просто перезапустите с `--full`, он продолжит с места сбоя.

## Языки программирования

Скрипт устанавливает 8 языков. Каждый — с обоснованием выбора.

### Java 25 (Adoptium)

| Характеристика | Значение |
|---------------|----------|
| Версия | Java 25 LTS |
| Метод установки | Adoptium API (прямая загрузка с GitHub CDN) |
| Fallback | `apt openjdk-25-jdk` |
| Дополнительно | SDKMAN → Gradle, Maven, Kotlin, jbang |
| Почему Adoptium | Надёжнее SDKMAN в WSL2. SDKMAN часто падает по DNS, Adoptium использует GitHub CDN — стабильно |
| Альтернативы | Oracle JDK (платная лицензия), OpenJDK (apt — старая версия), Corretto (Amazon, нет преимуществ перед Adoptium), GraalVM (для специфичных задач с native image) |

**Что дают дополнительные инструменты:**
- **Gradle** — система сборки (альтернатива Maven, быстрее, гибче)
- **Maven** — система сборки (альтернатива Gradle, стандарт в enterprise)
- **Kotlin** — язык от JetBrains, совместим с Java, лаконичнее
- **jbang** — запуск Java-скриптов без сборки (как Python)

### Node.js 22

| Характеристика | Значение |
|---------------|----------|
| Версия | Node.js 22 LTS |
| Метод установки | `n` (npx n 22) |
| Fallback | `apt nodejs` |
| Почему `n` | Позволяет легко переключать версии. `nvm` медленнее, `fnm` требует Rust, `n` — чище |
| Альтернативы | nvm (популярен, но медленный старт shell), fnm (быстрый, требует Rust), volta (хорош, но на Rust), asdf (универсальный, но медленный) |

**Почему Node.js 22, а не 24?**
Node.js 22 — LTS (долгосрочная поддержка до апреля 2027). Node 24 ещё не LTS. Для production-проектов LTS критичен.

### Python 3.12 + uv

| Характеристика | Значение |
|---------------|----------|
| Версия | Python 3.12 |
| Менеджер пакетов | uv (Astral) |
| Метод установки | `uv python install 3.12` |
| Fallback | `apt python3` |
| Почему uv | В 10–100 раз быстрее pip. Написан на Rust. Управляет и пакетами, и версиями Python |
| Альтернативы | pip (стандарт, медленный), poetry (управление зависимостями, медленный), pipenv (устарел), conda (тяжёлый, для data science) |

**Почему Python 3.12, а не 3.13?**
Python 3.12 — стабильный, все библиотеки совместимы. Python 3.13 ещё имеет проблемы совместимости с популярными пакетами (numpy, torch, etc).

### Go 1.24+

| Характеристика | Значение |
|---------------|----------|
| Версия | Авто-последняя (1.24+) |

| Fallback | `apt golang-go` |
| Почему прямая загрузка | `apt` даёт старую версию. Прямая загрузка — всегда свежая |
| Альтернативы | gvm (менеджер версий, нестабилен), asdf (медленный), snap (контейнеризация мешает) |

### Rust (rustup)

| Альтернативы | apt (устаревшая версия), snap (проблемы с путями), прямая загрузка (нет управления версиями) |

### Kotlin 2.1+

Устанавливается через SDKMAN. Современный язык от JetBrains, полностью совместим с Java, используется в Android-разработке и backend (Ktor, Spring).

Компилируемый язык, позиционируется как замена C. Включён как зависимость для некоторых инструментов (например, Bun написан на Zig).

**Что будет, если я не знаю какой-то язык?**

## MCP-серверы

MCP (Model Context Protocol) — это протокол, позволяющий AI-агенту взаимодействовать с внешними инструментами. Представьте: вы даёте AI задачу, а он **сам** читает ваши файлы, запускает браузер, ищет в базе данных, проверяет документацию библиотек.

Без MCP агент «слеп» — он видит только ваш запрос и свои предыдущие ответы. С MCP он «видит» весь проект.

Устанавливается 13 MCP-серверов (11 локальных + 2 удалённых). 9 активны по умолчанию, 4 отключены (включаются вручную).

| Пакет | `c7-mcp-server` |
| Команда | `c7-mcp-server` |
| Что делает | Даёт AI-агенту доступ к живой документации популярных библиотек |
| Когда полезен | Агент пишет код с Next.js, React, Prisma — он сам находит актуальную документацию |

### Отключённые по умолчанию

#### github — GitHub API

Отключён, потому что требует токен GitHub. Если включить — агент сможет создавать issues, PR, читать репозитории.

#### postgres — PostgreSQL

Отключён, потому что не у всех есть PostgreSQL. Если включить — агент сможет инспектировать схему БД, выполнять запросы.

Удалённый MCP от Sentry. Отключён — требует аккаунт Sentry. Полезен для команд с настроенным мониторингом.

Удалённый MCP от grep.app. Отключён — требует интернет. Позволяет искать примеры кода в миллионах репозиториев.

**Как включить/отключить MCP:**
В файле `~/.config/opencode/opencode.json` найдите нужный сервер в секции `mcp` и измените `"enabled": false` на `"enabled": true`.

## LSP-серверы

LSP (Language Server Protocol) — это то, что даёт вашему редактору:
- **Автодополнение** (начинаете писать — редактор предлагает варианты)
- **Подсветку ошибок** (красное подчёркивание если синтаксис неверный)
- **Навигацию** (прыжок к определению функции, поиск использований)
- **Рефакторинг** (переименование переменной во всём проекте)

Скрипт устанавливает 10 LSP-серверов (по одному на каждый язык + универсальные):

| LSP | Язык | Расширения | Что даёт |
|-----|------|-----------|----------|
| **gopls** | Go | `.go` | Официальный LSP от Google. Автодополнение, навигация, рефакторинг |
| **rust-analyzer** | Rust | `.rs` | Де-факто стандарт. Показывает типы, автодополнение, clippy-подсказки |
| **typescript-language-server** | TypeScript/JavaScript | `.ts`, `.tsx`, `.js`, `.jsx` | Автодополнение с типами, проверка ошибок |
| **pyright** | Python | `.py` | От Microsoft. Быстрее и строже чем pylance. Статическая типизация |
| **omnisharp** | C# | `.cs` | Стандартный LSP для .NET. Интеграция с проектами |
| **yaml-language-server** | YAML | `.yaml`, `.yml` | Валидация YAML, автодополнение схем (K8s, Docker Compose, CI/CD) |
| **marksman** | Markdown | `.md` | Навигация по заголовкам, проверка ссылок между .md файлами |
| **taplo** | TOML | `.toml` | Форматирование и валидация TOML (Cargo.toml, pyproject.toml) |
| **lua-language-server** | Lua | `.lua` | Для Neovim-конфигураций и скриптов |
| **zls** | Zig | `.zig` | LSP для Zig. Автодополнение, проверка ошибок |

**Почему именно эти 10:**
- Покрывают все 8 языков из скрипта
- Добавляют поддержку YAML (конфиги Docker, K8s) и TOML (конфиги Rust, Python)
- marksman — навигация по документации проекта
- lua — для тех, кто настраивает Neovim (популярный редактор в AI-сообществе)

**Альтернативы:** Существуют другие LSP для тех же языков (pylance вместо pyright, tsserver вместо typescript-language-server), но выбранные варианты — самые стабильные и быстрые.

## Плагины OpenCode

Плагины расширяют функциональность самого OpenCode. В отличие от MCP (внешние инструменты), плагины работают внутри OpenCode и влияют на его поведение.

### 1. opencode-codegraph (всегда активен)

| Характеристика | Значение |
|---------------|----------|
| Пакет | `opencode-codegraph` |
| Что делает | Автоматически обогащает каждый диалог графом вызовов проекта |
| На что влияет | Агент видит структуру кода, связи между функциями, точки входа |
| Без него | Агент анализирует код «вслепую» — только grep-поиск |
| Почему выбран | Единственный плагин, дающий структурное понимание кода |

### 2. opencode-dcp — Dynamic Context Pruning (сжатие контекста)

| Характеристика | Значение |
|---------------|----------|
| Пакет | `@tarquinen/opencode-dcp` |
| Конфигурация | compress: 20K–48K токенов с авто-компакцией. prune: защита 40K токенов, мин 20K |
| Что делает | Когда диалог становится слишком длинным — сжимает старые сообщения, сохраняя смысл. Защищает важные части от удаления |
| На что влияет | Позволяет вести длинные диалоги не теряя контекст. Экономит деньги (каждый токен стоит денег) |
| Без него | Контекст растёт → дороже → может превысить лимит модели |
| Альтернативы | Встроенная компрессия OpenCode (базовая), ручное управление контекстом |
| Почему выбран | Единственный плагин с настраиваемыми порогами компрессии |

### 3. opencode-stranger-danger — Защита от утечек

| Характеристика | Значение |
|---------------|----------|
| Пакет | `@gitdamnit/opencode-stranger-danger` |
| Что делает | Фильтрует PII и секреты из диалогов: ключи API, токены, пароли, email, телефоны |
| Что блокирует | Prompt-инъекции (попытки обмануть агента через входные данные) |
| На что влияет | Безопасность: ваши секреты не утекут в логи AI-провайдера |
| Почему выбран | Критичен если вы передаёте агенту production-конфиги или код с секретами |

### 4. opencode-damage-control — 144 защитных паттерна

| Характеристика | Значение |
|---------------|----------|
| Пакет | `opencode-damage-control` |
| Что делает | Блокирует опасные команды до их выполнения |
| Заблокировано | `rm -rf /` (удаление системы), `DROP TABLE` (удаление БД), `terraform destroy` (разрушение инфраструктуры) |
| Заблокированы пути | `~/.ssh`, `~/.aws`, `.env` — агент не может читать/писать в эти директории |
| Аудит | Защищённый лог на 365 дней. Нельзя подделать или удалить |
| На что влияет | Предотвращает катастрофические ошибки. Даже если агент «галлюцинирует» опасную команду — она не выполнится |
| Альтернативы | Ручной code review (медленно), chroot/контейнеризация (сложно настроить) |
| Почему выбран | 144 паттерна покрывают большинство известных векторов атак |

### 5. opencode-auto-fallback — Автопереключение провайдеров

| Характеристика | Значение |
|---------------|----------|
| Пакет | `opencode-auto-fallback` |
| Что делает | Если основной провайдер (DeepSeek) недоступен — автоматически переключает на следующий (OpenCode Go → xAI → MiMo → Moonshot → MiniMax) |
| На что влияет | Непрерывность работы. Даже если DeepSeek «лёг» — агент продолжит работать |
| Без него | Ошибка от провайдера = остановка работы |

### 6. opencode-lazy-loader — Отложенная загрузка

| Характеристика | Значение |
|---------------|----------|
| Пакет | `opencode-lazy-loader` |
| Что делает | Не загружает все плагины сразу при старте. Загружает только когда они реально нужны |
| На что влияет | Скорость запуска OpenCode. Без него — все плагины грузятся сразу, замедляя старт |

### 7. open-orchestra — Оркестрация агентов

| Характеристика | Значение |
|---------------|----------|
| Пакет | `open-orchestra` |
| Что делает | Координирует работу нескольких агентов. Один анализирует, другой пишет код, третий проверяет |
| Когда полезен | Сложные проекты где нужна параллельная работа агентов |
| Альтернативы | LangChain (тяжелее, для Python), CrewAI (другой подход), ручная координация |

**Итоговая цепочка безопасности плагинов:**
```
stranger-danger → фильтрует опасный ввод
       ↓
damage-control → блокирует опасные команды
       ↓
codegraph → даёт структурное понимание кода
       ↓
dcp → сжимает контекст при переполнении
       ↓
auto-fallback → переключает провайдера при сбое
```

## Провайдеры AI

Скрипт настраивает 6 AI-провайдеров. Основной — DeepSeek, остальные — резервные и для специфичных задач.

### DeepSeek (основной)

| Модель | Назначение | Цена (вход / выход за 1M токенов) |
|--------|-----------|----------------------------------|
| `deepseek-v4-pro` | Основная: reasoning, 64K вывод | ≈$2.00 / $8.00 |
| `deepseek-v4-flash` | Быстрая: фоновые задачи, поиск | ≈$0.14 / $0.56 |

**Почему DeepSeek:**
- Лучшее соотношение цена/качество на рынке (июнь 2026)
- DeepSeek V4 Pro побеждает GPT-5 и Claude 4 в бенчмарках кода
- Flash-версия в 15 раз дешевле Pro — для рутинных задач
- Китайский регулятор не блокирует API для РФ (в отличие от OpenAI)
- 64K вывод vs 16K у конкурентов — можно генерировать большие файлы за один запрос

**Альтернативы:**
- OpenAI (GPT-5): дороже в 3–5 раз, блокировки для РФ
- Anthropic (Claude 4): отличный код, но дороже и тоже блокировки
- Google (Gemini 3): хорош, но API нестабилен
- Qwen (Alibaba): дешевле, но слабее в сложных задачах

## GPU / Локальные LLM

Опциональный блок. Устанавливается если выбрать `y` в интерактивном режиме на вопросе «Local LLM runtimes?» или через `dev install llm`.

### Ollama

| Характеристика | Значение |
|---------------|----------|
| Что это | Движок для запуска LLM локально (на вашем компьютере) |
| Требования | CPU (любой) или GPU (NVIDIA/AMD) |
| Модель по умолчанию | qwen3:1.8b (лёгкая, ~1GB RAM) |
| Использование | `ollama run qwen3:14b`, `ollama pull deepseek-r1:8b` |
| Плюсы | Бесплатно, приватно (данные не уходят в облако), офлайн |
| Минусы | Медленнее облака, качество ниже чем у DeepSeek V4 Pro, требует RAM/VRAM |

**Альтернативы:** LM Studio (GUI, проще для новичков), llama.cpp (сложнее, больше контроля), GPT4All (проще, но медленнее)

**Когда использовать:**
- Нет интернета
- Конфиденциальные данные (нельзя отправлять в облако)
- Эксперименты и обучение
- Простые задачи (автодополнение, форматирование)

### vLLM

Высокопроизводительный сервер инференса. Требует NVIDIA GPU + CUDA.

**Когда использовать:** Если у вас мощная видеокарта (RTX 4090, A100) и нужно обслуживать несколько пользователей/агентов одновременно.

**Альтернативы:** TGI (HuggingFace, тоже требует GPU), llama.cpp (CPU-first)

### SGLang

Эффективная структурная генерация. Требует GPU.

**Когда использовать:** Нужен контролируемый вывод (JSON, SQL, код с гарантированной структурой).

### Open WebUI

| Характеристика | Значение |
|---------------|----------|
| Что это | Чат-интерфейс (как ChatGPT) для Ollama |
| Адрес | http://localhost:3300 |
| Установка | Docker (автоматически) |
| Альтернативы | LM Studio (свой UI), Ollama Web UI (проще), Anything LLM (больше фич, сложнее) |

### LlamaEdge

Лёгкий runner на WasmEdge. CPU-friendly.

**Когда использовать:** Слабые машины без GPU. Позволяет запускать небольшие модели (до 7B) даже на 8GB RAM.

## Безопасность

Скрипт реализует **эшелонированную защиту** — несколько уровней, каждый защищает от своего класса угроз.

### Уровень 1: Защита секретов

- **Все API-ключи** хранятся в `~/.config/opencode/secrets.env`
- Файл имеет права **600** (только владелец может читать)
- Ключи **никогда не пишутся** в `.bashrc` или `.zshrc`
- При генерации `.env` файла автоматически выставляются правильные права

### Уровень 2: Фильтрация ввода (stranger-danger)

- Фильтрует PII (персональные данные) из диалогов
- Блокирует prompt-инъекции (попытки заставить агента сделать вредоносное действие)
- Предотвращает случайную отправку секретов AI-провайдеру

### Уровень 3: Блокировка опасных команд (damage-control)

144 защищённых паттерна блокируют:
- `rm -rf /` — удаление всей системы
- `DROP TABLE` / `DROP DATABASE` — уничтожение базы данных
- `terraform destroy` — разрушение облачной инфраструктуры
- Доступ к `~/.ssh`, `~/.aws`, `.env` — кража ключей

Аудит-лог пишется 365 дней, защищён от подделки.

### Уровень 4: Сканеры уязвимостей

**Trivy** — сканирует Docker-образы, зависимости, IaC на известные уязвимости (CVE).

| Альтернативы | Snyk (платный), Grype (медленнее), Clair (сложнее в установке) |

**Qodana** — статический анализатор кода от JetBrains. Находит баги, уязвимости, нарушения код-стиля.

| Альтернативы | SonarQube (тяжелее, требует сервер), CodeQL (сложный), Semgrep (быстрее, меньше правил) |

### Уровень 5: WSL2-безопасность

- Отключён Windows PATH в WSL2 (`appendWindowsPath=false`) — вирусы Windows не могут запуститься из WSL
- Systemd как init-система — современная изоляция процессов
- Авто-очистка кэша памяти (dropCache) — предотвращает утечки данных между сессиями

## Инфраструктура

Скрипт генерирует `docker-compose.yml` с 8 сервисами для локальной разработки:

| Сервис | Версия | Порт | Зачем |
|--------|--------|------|-------|
| **PostgreSQL** | 18 Alpine | 5432 | Основная база данных. Alpine-версия — минимальный образ |
| **MongoDB** | 8 | 27017 | Документная БД. Для проектов с гибкой схемой данных |
| **Redis** | 7 Alpine | 6379 | Кэш, очереди, сессии. Ускоряет приложение в 10–100 раз |
| **Kafka** | Confluent 7.5.0 | 9092 | Потоковая обработка данных. Для микросервисов и событийной архитектуры |
| **MinIO** | latest | 9000/9001 | S3-совместимое хранилище файлов. Локальный аналог AWS S3 |
| **WireMock** | 3.5.4 | 8080 | Моки HTTP API. Имитирует ответы внешних сервисов для тестов |
| **Keycloak** | 26.0 | 8081 | Аутентификация и авторизация. Локальный аналог Auth0/Okta (профиль: auth) |

**Почему именно эти сервисы:**
Это — минимальный набор для современной веб-разработки. PostgreSQL — стандарт для реляционных данных. Redis — стандарт для кэширования. Kafka — стандарт для событийной архитектуры. MinIO — стандарт для локальной разработки с S3. WireMock — стандарт для мокирования API в тестах.

## Конфигурация

### opencode.json

Главный конфигурационный файл: `~/.config/opencode/opencode.json`

```json
{
  "model": "deepseek/deepseek-v4-pro",    // основная модель
  "small_model": "deepseek/deepseek-v4-flash", // для фоновых задач
  "compaction": {
    "auto": true,          // авто-сжатие контекста
    "prune": true,         // удаление неважного
    "tail_turns": 2,       // сохранять последние 2 хода
    "reserved": 10000       // резерв токенов
  },
  "tool_output": {
    "max_lines": 2000,     // макс строк вывода инструментов
    "max_bytes": 51200      // макс байт вывода
  },
  "watcher": {
    "ignore": ["node_modules/**", "dist/**", ".git/**"]
  },
  "experimental": {
    "mcp_timeout": 30000,  // таймаут MCP (30 сек)
    "openTelemetry": true   // телеметрия
  }
}
```

- Java? **n** (если не знаете что это)

- .NET? **n**

**Итог:** Node.js + Python + OpenCode + MCP + LSP. Минимальный набор для веб-разработки с AI.

## Решение проблем

### Скрипт упал с ошибкой

```bash
# Проверить что сломалось
bash setup.sh --health

# Перезапустить (продолжит с места сбоя)
bash setup.sh --full
```

### OpenCode не запускается

```bash
# Проверить конфиг на валидность
cat ~/.config/opencode/opencode.json | python3 -m json.tool

# Перегенерировать
bash setup.sh --fix-config

# Запустить без проекта (проверить что сам OpenCode работает)
OPENCODE_DISABLE_PROJECT_CONFIG=1 opencode
```

### MCP-сервер не работает

```bash
# Проверить установку
npm list -g | grep mcp

# Переустановить все MCP
bash setup.sh --reinit
```

### Проблемы с DNS / интернетом (РФ/СНГ)

Скрипт автоматически добавляет Yandex DNS (77.88.8.8) и Cloudflare (1.1.1.1) в `/etc/resolv.conf`. Если не помогло:

```bash
# Добавить вручную
echo "nameserver 77.88.8.8" | sudo tee -a /etc/resolv.conf
echo "nameserver 1.1.1.1" | sudo tee -a /etc/resolv.conf
```

### WSL2: сеть не работает после сна

Скрипт автоматически чинит WSL2 DNS и прокси. Если не помогло — перезапустить WSL2:
```powershell
wsl --shutdown
```

### Терминал глючит (zsh)

```bash
bash setup.sh --fix-zshrc
```

### Хочу удалить конкретный компонент

```bash
dev remove java      # удалить Java
dev remove docker    # удалить Docker
dev list             # посмотреть что установлено
```

### Хочу обновить всё до последних версий

```bash
dev update           # обновить инструменты
dev self-update      # обновить сам скрипт
```
